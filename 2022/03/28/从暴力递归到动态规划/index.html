从暴力递归到动态规划

无标签
发布日期:   2022-03-28

一道例题

假设有排成一行的N个位置，记为1到N,N一定大于或等于2
开始时机器人在其中的M位置上（M一定是1到N中的一个）
如果机器人来到1位置，那么下一步只能往右来到2位置;
如果机器人来到N位置，那么下一步只能往左来到N-1位置;
如果机器人来到中间位置，那么下一步可以往左走或者往右走;
规定机器人必须走K步，最终能来到P位置（P也是1~N中的一个）的方法有多少种给定四个参数N、M、K、P，返回方法数。

方法1：递归尝试

思路

两个变量：cur表示当前位置，rest表示剩余可走步数，尝试方法返回在当前cur、rest情况下，有多少种走法到达P；

  1. rest为0的情况下，如果cur到达p位置，那么结算为1种走法；否则这种走法行不通，结算为0；
  2. 如果cur到了1（最左侧），那么只能在cur=2，可走rest-1的情况下去尝试；
  3. 如果cur到了N（最右侧），那么只能在cur=N-1，可走rest-1的情况下去尝试；
  4. 其他情况下（中间位置），那么既可以向左，又可以向右尝试，走法是两种情况的总和。
//固定参数N，P，可变参数cur，rest,返回在当前cur、rest情况下，有多少种走法到达P；
public int walk(int N,int cur,int rest,int P){
    if (rest == 0){
        return cur == P ? 1 : 0;
    }
    if (cur == 1){
        return walk(N,2,rest-1,P);
    }
    if (cur == N) {
        return walk(N, N - 1, rest - 1, P);
    }
    return walk(N, cur - 1, rest - 1, P) + walk(N, cur + 1, rest - 1, P);
}

复杂度

空间：使用了递归栈，递归树最大深度为K，所以为O(K)
时间：递归树节点的规模在2^K，所以为O(2^K)

暴力递归的不足

暴力递归可能存在大量重复计算，我们把上面的函数简化为f(cur, rest)，目标P为4.
举个例子，要计算f(3,3)的值。
要计算f(3,3)的值，就要先计算f(2,2)和f(4,2);
要计算f(2，2)的值，就要先计算f(1,1)和f(3,1);
要计算f(4,2)的值，就要先计算f(3,1)和f(5,1);

从递归树中，可以很清楚地看到，f(3,1)在两个分支中被重复计算。
那么有没有什么方法能把其他分支计算过的结果缓存起来呢？
答案是动态规划，也就是用数组保存。

改进方法1：动态规划（记忆化搜索）

思路

前面提到的f(3,1)的值，我们把他保存在dp[3][1]的位置当中，以给其他分支使用。
cur可能的变化范围是1…N，rest可能的变化范围是0…K，因此开辟[N+1][K+1]大小的数组，这个数组可以把递归过程的所有返回值装下。

  • 主方法中开辟dp数组，并且初始化
  • walk方法参数中，加一个dp数组
  • 在walk方法中
  1. 如果发现dp中的值不为-1，说明之前保存过状态，直接返回，不再走后面的递归；
  2. 其他分支，相比原本的暴力递归直接返回，变成了先设置dp值，再返回的过程。
public int walkCache(int N,int cur,int rest,int P,int[][] dp){
    if (dp[cur][rest]!=-1){
        return dp[cur][rest];
    } //如果发现dp中的值不为-1，说明之前保存过状态，直接返回，不再走后面的递归；
    if (rest == 0){
        dp[cur][rest]=cur == P ? 1 : 0;
        return dp[cur][rest];  //先保存，再返回
    }
    if (cur == 1){
        dp[cur][rest] = walkCache(N, 2, rest - 1, P, dp);
        return dp[cur][rest]; //先保存，再返回
    }
    if (cur == N) {
        dp[cur][rest]=walkCache(N, N - 1, rest - 1, P,dp);
        return dp[cur][rest]; //先保存，再返回
    }
    dp[cur][rest]=walkCache(N, cur - 1, rest - 1, P,dp) + walkCache(N, cur + 1, rest - 1, P,dp);
    return dp[cur][rest]; //先保存，再返回
}

public int waysCache(int N,int M,int K,int P){
    int[][] dp = new int[N + 1][K + 1];
    for (int row = 0; row <=N ; row++) {
        for (int col = 0; col <=K ; col++) {
            dp[row][col] = -1;  //初始化，所有信息都不知道
        }
    }
    return walkCache(N,M,K,P,dp);
}

复杂度

空间：使用了递归栈，递归树最大深度为K，又申请了一张表，所以为O(N*K)
时间：逐步填满了整张表，所以为O(N*K)

改进方法2：经典动态规划

在改进方法1中，我们使用了动态规划优化了时间复杂度，但是这种方法内部看起来还是包含了递归；
经典的动态规划中，不出现递归，而是直接找dp[i][j]和它上下左右的依赖关系，一步步把整张dp填满。

思路

从改进方法1或暴力递归中，我们可以看出：

  • 在第0列上，除了dp[P][0]=1,其他的dp[…][0]均为0；
  • 在第一行上，dp[1][rest]=dp[2][rest-1];即除了最左的格子，其他格子都依赖于左下角的格子
  • 在中间行上，dp[cur][rest]=dp[cur-1][rest-1]+dp[cur+1][rest-1];;即除了最左的格子，其他格子都依赖于左上角和左下角的格子
  • 在最后一行上，dp[N][rest]=dp[N-1][rest-1];即除了最左的格子，其他格子都依赖于左上角的格子
    public int waysCache2(int N,int M,int K,int P){
  int\[][] dp = new int[N + 1][K + 1];
//        for (int row = 1; row <=N ; row++) {
//            dp[row][0]=0;  //可省略，因为初始化的数组默认是0
//        }
  dp[P][0]=1;
  //以下循环可将row重命名为cur，col重命名为rest
  for(int col=1;col<=K;col++){
      dp[1][col]=dp[2][col-1];  //第一行特殊处理
      for (int row=2;row<=N-1;row++){
          dp[row][col]=dp[row-1][col-1]+dp[row+1][col-1];
      }
      dp[N][col]=dp[N-1][col-1]; //最后一行特殊处理
  }
  return dp[M][K];
}

    空间：申请了一张表，所以为O(N*K)
    时间：逐步填满了整张表，所以为O(N*K)
