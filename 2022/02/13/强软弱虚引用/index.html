强软弱虚引用

无标签
发布日期:   2022-02-13

四种引用

强引用、软引用、弱引用、虚引用。
C/C++对于对象的内存分配和回收，需要程序员来负责。而JAVA对内存的分配和回收，由JVM管理。判断一个对象回收与否，使用根可达算法，本质上还是看有没有引用指向这个（或这些）对象。
四种引用，是为了设定对象被回收的时机，从而更好地管理对象的生命周期。

强引用

这是Java使用的最多的引用，形如M m = new M();
这种引用，除非不再有引用指向对象，否则对象永远不会被回收。
我们重写M类的finalize方法，以便更好地观察JVM回收掉了对象。

public class M {
    @Override
    protected void finalize() throws Throwable {
        System.out.println("finalized");
    }
}

将m设置为null，此时没有引用指向内存里的对象，执行垃圾回收，finalized被打印

public static void main(String[] args) throws IOException {
    M m = new M();
    m = null;  //断开引用
    System.gc(); //回收对象
}

软引用

当内存不足时，JVM会把软引用指向的对象回收掉。
这是创建软引用的方法，这里创建了一个强引用m，指向了了一个SoftReference对象，这个SoftReference对象，又通过软引用指向了o。要取到对象o，就调用get()方法。

SoftReference<Object> m = new SoftReference<>(new Object);
System.out.println(m.get());

既然内存不足时，JVM会把软引用指向的对象回收掉。那么我们先把堆内存设置为20M。以下设置表示最少20，最多20，也就是20。

-Xms20M -Xmx20M

我们先创建一个指向10M大小数组的软引用，再创建一个12M大小的数组，那么就会出现堆内存不足的异常。

public static void main(String[] args) {

    SoftReference<byte[]> m = new SoftReference<>(new byte[1024*1024*10]);
//        m = null;
    System.out.println(m.get());
    System.gc();
    try {
        Thread.sleep(500);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    System.out.println(m.get());

    //再分配一个数组，heap将装不下，这时候系统会垃圾回收，先回收一次，如果不够，会把软引用干掉
    byte[] b = new byte[1024*1024*12];
    System.out.println(m.get());
}

弱引用

如果弱引用包裹的对象没有被强引用指向，那么只要发生GC时，对象就会被回收。

public static void main(String[] args) {
    byte[] myByteArray = new byte[1];
    WeakReference weakReference = new WeakReference(myByteArray);
    System.out.println(weakReference.get());
    myByteArray = null;
    System.gc();
    System.out.println(weakReference.get());
}

虚引用

虚引用的特点是，无法通过get来访问。以下代码得到的结果是null。

 ReferenceQueue queue = new ReferenceQueue();
PhantomReference<byte[]> reference = new PhantomReference<byte[]>(new byte[1], queue);
System.out.println(reference.get());

以下例子中，第一个线程往集合里面塞数据，随着数据越来越多，会发生GC。 第二个线程不断从queue里面拿数据，如果拿出来的数据不是null，就打印出来。

private static final List<Object> LIST = new LinkedList<>();
private static final ReferenceQueue<M> QUEUE = new ReferenceQueue<>();
public static void main(String[] args) {
    PhantomReference<M> phantomReference = new PhantomReference<>(new M(), QUEUE);
    new Thread(() -> {
        while (true) {
            LIST.add(new byte[1024 * 1024]);
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
                Thread.currentThread().interrupt();
            }
            System.out.println(phantomReference.get());
        }
    }).start();

    new Thread(() -> {
        while (true) {
            Reference poll = QUEUE.poll();
            if (poll != null) {
                System.out.println("--- 虚引用对象被jvm回收了 ---- " + poll);
            }
        }
    }).start();
}

— 虚引用对象被jvm回收了 —- java.lang.ref.PhantomReference@e02eac7

虚引用的作用是，管理堆外内存。
